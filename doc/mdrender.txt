1. Introduction						*mdrender-intro*

vim-mdrender renders Markdown files visually inside Vim using two display
layers that cooperate without ever modifying buffer content:

  Syntax / conceal layer ~
    Inline markers (**,*,`,~~,[]) are hidden using Vim's built-in |conceal|
    feature. The surrounding text receives highlight groups (bold, italic,
    etc.) via `syntax region ... concealends` rules.

  Text-property layer ~
    Block-level decorations — code-block backgrounds, horizontal rules
    rendered as a full-width line, table header highlighting — are applied
    via |prop_add()| with range highlighting and virtual text.

The buffer content is never modified. Toggling the plugin off restores the
original display exactly.

==============================================================================
2. Requirements						*mdrender-requirements*

  - Vim 9.0 or later (Vim9 script, virtual text in |prop_add()|)
  - |+textprop| compiled in: `has('textprop')` must return 1
  - |+conceal|  compiled in: `has('conceal')`  must return 1

Check both at once: >vim
    echo has('textprop') && has('conceal')
<
Both features are included in the default `+huge` build on all platforms.

==============================================================================
3. Installation						*mdrender-install*

vim-plug: >vim
    Plug 'your-user/vim-mdrender'
<
Vim built-in packages: >sh
    git clone https://github.com/your-user/vim-mdrender \
        ~/.vim/pack/plugins/start/vim-mdrender
    vim -u NONE -c 'helptags ALL' -c q
<
The plugin activates automatically for any buffer with `filetype=markdown`.

==============================================================================
4. Quick start						*mdrender-quickstart*

Open a Markdown file — rendering is on by default.

Toggle off and on: >vim
    :MdRenderToggle
<
Disable auto-enable and control it manually: >vim
    " In vimrc
    let g:mdrender_enabled = 0

    " Then in a markdown buffer:
    :MdRenderEnable
<
Opt specific elements out: >vim
    let g:mdrender_elements = {'tables': v:false, 'hr': v:false}
<
Apply a built-in theme: >vim
    let g:mdrender_theme = 'dark'    " or 'light', 'minimal', 'auto'
<
Override a highlight colour (beats any theme): >vim
    autocmd ColorScheme * hi MdH1 guifg=#ff6688 gui=bold
<
==============================================================================
5. Commands						*mdrender-commands*

:MdRenderEnable					*:MdRenderEnable*
	Enable rendering in the current buffer. Sets |conceallevel| to 2,
	installs syntax groups, registers text properties, and performs the
	first render pass.

:MdRenderDisable				*:MdRenderDisable*
	Disable rendering. Removes all plugin syntax groups, text properties,
	and highlight definitions. Restores the user's original |conceallevel|.

:MdRenderToggle					*:MdRenderToggle*
	Toggle rendering on or off in the current buffer.

6. Configuration					*mdrender-config*

All options are global variables. Set them in your vimrc.

						*g:mdrender_enabled*
g:mdrender_enabled (default: 1) ~
	Set to 0 to prevent the plugin from auto-activating on markdown files.
	You can still call |:MdRenderEnable| manually. >vim
	    let g:mdrender_enabled = 0
<
						*g:mdrender_conceallevel*
g:mdrender_conceallevel (default: 2) ~
	The |conceallevel| value applied while rendering is active.
	  0  — no concealment (rendering has no visual effect)
	  1  — concealed text replaced by a single substitution character
	  2  — concealed text completely hidden (recommended)
	  3  — concealed text hidden including the |cchar| replacement >vim
	    let g:mdrender_conceallevel = 1
<
						*g:mdrender_max_file_size*
g:mdrender_max_file_size (default: 204800) ~
	Files larger than this many bytes are skipped. Large files make the
	parser slow and the syntax engine expensive. Default is 200 KiB. >vim
	    let g:mdrender_max_file_size = 524288   " 512 KiB
<
						*g:mdrender_debounce_ms*
g:mdrender_debounce_ms (default: 150) ~
	Milliseconds of typing inactivity before a re-render is triggered.
	Lower values feel more responsive; higher values reduce CPU usage
	during fast typing. >vim
	    let g:mdrender_debounce_ms = 300
<
						*g:mdrender_cursor_reveal*
g:mdrender_cursor_reveal (default: 1) ~
	When 1 (default), the cursor line always shows raw Markdown source
	in all modes. This is achieved by setting |concealcursor| to empty,
	which is Vim's built-in mechanism for cursor-line reveal. Set to 0
	to conceal the cursor line as well. >vim
	    let g:mdrender_cursor_reveal = 0
<
						*g:mdrender_render_pad*
g:mdrender_render_pad (default: 50) ~
	Number of lines above and below the visible window to pre-render.
	A higher value reduces flicker when scrolling fast at the cost of
	parsing and property-insertion work on invisible lines. >vim
	    let g:mdrender_render_pad = 100
<
						*g:mdrender_elements*
g:mdrender_elements (default: all enabled) ~
	A dict controlling which Markdown elements are rendered. Keys not
	present in the user's dict default to enabled. >vim
	    let g:mdrender_elements = {
	    \   'tables':  v:false,
	    \   'hr':      v:false,
	    \   'tasks':   v:false,
	    \ }
<
	Available keys:
	    headings, bold, italic, code_inline, code_block, links,
	    strikethrough, blockquotes, hr, lists, tasks, tables

						*g:mdrender_bullet_chars*
g:mdrender_bullet_chars (default: ['•', '◦', '▪']) ~
	Three Unicode characters used as bullet replacements for unordered
	list items at indent depths 0, 1, and 2+. >vim
	    let g:mdrender_bullet_chars = ['-', '·', '·']
<
						*g:mdrender_theme*
g:mdrender_theme (default: 'auto') ~
	Choose a colour scheme for the MdXxx highlight groups. Accepts a
	string naming a built-in or user-defined theme, or an inline dict.

	Built-in theme names: ~
	  'auto'    Current behaviour: every group links to a colorscheme
		    semantic group (Title, Statement, ...) so colours follow
		    whatever theme is active. This is the default.
	  'dark'    Opinionated dark palette (One Dark-inspired). Requires
		    a dark background and 'termguicolors'.
	  'light'   Opinionated light palette (GitHub-inspired). Requires
		    a light background and 'termguicolors'.
	  'minimal' Style-only: no colour at all. Heading levels are
		    distinguished by bold/italic/underline combinations only.

	Named theme: >vim
	    let g:mdrender_theme = 'dark'
	<
	User theme files ~

	Any name that is not a built-in triggers a file search. The plugin
	looks for `mdrender/themes/<name>.vim` in every directory in
	'runtimepath'. The file must assign the theme dict to the global
	variable `g:mdrender_theme_def`: >vim

	    " Example: ~/.config/vim/mdrender/themes/ocean.vim
	    let g:mdrender_theme_def = {
	    \   'MdH1': 'guifg=#96cbfe gui=bold ctermfg=75 cterm=bold',
	    \   'MdH2': 'guifg=#7ec8e3 gui=bold ctermfg=74 cterm=bold',
	    \   'MdH3': 'guifg=#99d1f5 gui=bold ctermfg=117 cterm=bold',
	    \ }
	<
	    " Then in vimrc:
	    let g:mdrender_theme = 'ocean'
	<
	The plugin clears `g:mdrender_theme_def` immediately after reading it.
	Any group absent from the dict falls through to the 'auto' links, so
	partial theme files are valid.

	Third-party plugins can ship themes by placing them under
	`mdrender/themes/` anywhere in 'runtimepath'.

	Inline dict ~

	Keys are MdXxx group names; values are the attribute string you would
	write after the group name in :highlight: >vim
	    let g:mdrender_theme = {
	    \   'MdH1': 'guifg=#ff6688 gui=bold ctermfg=210 cterm=bold',
	    \   'MdH2': 'guifg=#ffaa33 gui=bold ctermfg=215 cterm=bold',
	    \ }
	<
	Overrides and reload ~

	The theme is applied via |hi-def| so explicit user overrides always
	win: a |:highlight| command for any MdXxx group that runs after
	plugin initialisation (e.g. in a |ColorScheme| autocmd) takes
	precedence. >vim
	    autocmd ColorScheme * hi MdH1 guifg=#ff0000 gui=bold
	<
	To switch themes at runtime, change the variable then reload: >vim
	    :doautocmd ColorScheme
	<

7. Highlight groups					*mdrender-highlights*

When |g:mdrender_theme| is 'auto' (the default), all groups link to
standard Vim semantic groups so they automatically follow the active
colorscheme — no hardcoded colours. Named themes ('dark', 'light',
'minimal') replace these links with fixed colour or style definitions.

The plugin re-runs highlight setup on every |ColorScheme| event so groups
survive theme switches (most colorschemes call |:hi-clear| on load).

Override any group after your colorscheme loads — this beats the theme: >vim
    autocmd ColorScheme * hi MdH1 guifg=#ff6688 gui=bold
<
	Group		Links to	Element
	─────────	──────────	──────────────────────────────────
	MdH1		Title		Heading level 1
	MdH2		Statement	Heading level 2
	MdH3		Type		Heading level 3
	MdH4		Special		Heading level 4
	MdH5		Identifier	Heading level 5
	MdH6		Comment		Heading level 6
	MdBold		(bold only)	Bold text (**text**)
	MdItalic	(italic only)	Italic text (*text*)
	MdBoldItalic	(bold+italic)	Bold and italic (***text***)
	MdStrike	(strikethrough)	Strikethrough (~~text~~)
	MdCodeInline	String		Inline code span (`code`)
	MdCodeBlock	CursorLine	Fenced code block background
	MdCodeLang	Comment		Language tag on fenced blocks
	MdLink		Underlined	Link label text ([label](url))
	MdLinkDelim	NonText		Concealed link delimiters
	MdBlockquote	Comment		Blockquote body text
	MdBlockquoteMark Special	Blockquote ┃ bar character
	MdHRule		NonText		Horizontal rule virtual text
	MdListBullet	Special		Unordered list bullet (•)
	MdTaskTodo	Todo		Unchecked task (☐)
	MdTaskDone	Comment		Checked task (☑)
	MdTableBorder	NonText		Table separator row
	MdTableHeader	Statement	Table header row

Note: MdBold, MdItalic, MdBoldItalic, and MdStrike apply only a text
style (gui=bold etc.) with no colour change, so emphasis text stays the
same colour as surrounding normal text.

8. Rendered elements					*mdrender-elements*

Headings ~
	ATX headings # through ######. The leading `# ` prefix is concealed
	and the rest of the line is highlighted with MdH1–MdH6. Setext-style
	headings (underlined with = or -) are not currently rendered.

Bold / Italic / Bold-Italic ~
	**bold**, *italic*, ***bold-italic***. The delimiter characters are
	concealed via |concealends|. Both `**`/`__` (bold) and `*`/`_`
	(italic) marker styles are supported.

Inline code ~
	`code spans`. The surrounding backticks are concealed; the body
	text is highlighted with MdCodeInline.

Strikethrough ~
	~~text~~. Delimiters concealed; body highlighted with MdStrike.

Links ~
	[label](url). The `[`, `]`, `(`, url, and `)` are all concealed,
	leaving only the label visible. The label is highlighted with MdLink.
	Image syntax ![alt](url) is treated as a plain link for now.

Blockquotes ~
	Lines beginning with `> `. The `> ` prefix is concealed and replaced
	with a ┃ bar character (MdBlockquoteMark highlight).

Horizontal rules ~
	Lines matching `---`, `***`, or `___` (three or more characters,
	optional spaces). The line content is concealed and replaced with a
	`─` line spanning the window width via a text property.

Fenced code blocks ~
	Triple-backtick (```) and triple-tilde (~~~) fences. The fence lines
	are concealed; body lines receive a MdCodeBlock background highlight
	via text properties. An optional language tag on the opening fence
	is concealed with the MdCodeLang highlight.

Lists ~
	Unordered: `-`, `*`, `+` bullets are replaced with • (depth 0),
	◦ (depth 1), or ▪ (depth 2+). Configurable via |g:mdrender_bullet_chars|.
	Ordered: numbered items are not currently re-rendered.

Task lists ~
	- [ ] todo →  ☐  (MdTaskTodo)
	- [x] done →  ☑  (MdTaskDone)

Tables ~
	The row above the first `|---|---|` separator is highlighted as a
	header (MdTableHeader). The separator row itself is highlighted with
	MdTableBorder. Column pipe characters are not currently replaced.

9. Architecture notes					*mdrender-architecture*

The plugin is split across five modules under autoload/mdrender/:

  config.vim    Central accessor for all g:mdrender_* globals. Every other
		module calls Cfg.Get() rather than accessing g: directly.

  highlight.vim Defines all MdXxx highlight groups using `hi def`. Setup()
		applies the active theme first (built-in palette, user file from
		runtimepath, or inline dict), then falls back to `hi def link`
		for any group the theme did not cover. Teardown() clears all
		groups. Both are idempotent — safe to call multiple times.

  parser.vim    ParseRange(bufnr, top, bot) classifies lines into typed
		tokens using a single-pass state machine. The only state
		between lines is whether we are inside a fenced code block.
		Inline elements are NOT parsed here — they are handled by
		Vim's syntax engine.

  conceal.vim   Installs syntax rules (syntax match / syntax region) using
		Vim's |conceal| and |concealends| features for inline elements.
		Rules are installed once per buffer in Enable() and require no
		per-render work — Vim evaluates them automatically on redraw.

  props.vim     Manages text properties (|prop_add|, |prop_remove|) for
		block-level decorations that need virtual text or range
		backgrounds. Re-applied on every render pass for the visible
		range ± render_pad lines.

  mdrender.vim  Orchestrator. Holds per-buffer state (_state dict), wires
		autocmd events to render cycles, implements the debounce
		timer, and manages the Enable/Disable lifecycle.
